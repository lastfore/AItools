# 聊天、提问与转换 - 何种场景用何种工具？

Open Notebook 提供多种与研究材料交互的方式。理解何时使用哪一种，是高效使用系统的关键。

---

## 三种交互模式

### 1. CHAT（聊天）- 手动上下文的对话式探索

**是什么：** 与 AI 就所选来源进行对话。

**流程：**
```
1. 你选择要包含的来源（「在上下文中」）
2. 你提出问题
3. AI 仅使用这些来源作答
4. 你提出后续问题（上下文保持不变）
5. 你更换来源或上下文级别，然后继续
```

**上下文管理：** 你显式选择 AI 可见的来源。

**对话式：** 多轮问题，共享历史。

**示例：**
```
你：[选择来源："paper1.pdf"、"research_notes.txt"]
    [设置上下文：paper1 为全内容，notes 为摘要]

你："这些来源的主要论点是什么？"
AI："论文 1 主张 X [引用]。你的笔记强调 Y [引用]。"

你："它们有何不同？"
AI："论文 1 聚焦 X [引用]，而你的笔记突出 Y [引用]……"

你：[现在选择不同来源]

你："与这一其他视角相比如何？"
AI："这一新来源采取了不同思路……"
```

**最适合：**
- 用特定来源探索聚焦主题
- 需要对话（多轮来回提问）
- 已知哪些来源重要
- 希望对发送给 AI 的内容有严格控制

---

### 2. ASK（提问）- 自动化综合搜索

**是什么：** 提出一个复杂问题，系统自动查找相关内容。

**流程：**
```
1. 你提出一个综合性问题
2. 系统分析问题
3. 系统自动搜索你的来源
4. 系统检索相关片段
5. 系统综合所有结果生成答案
6. 你得到一份详细答案（非对话式）
```

**上下文管理：** 自动。系统判断何为相关。

**非对话式：** 一问一答。无后续追问。

**示例：**
```
你："这些论文如何比较其对齐（alignment）的方法？
     各自推荐什么？"

系统：
  - 将问题拆解为搜索策略
  - 在所有来源中搜索对齐方法
  - 在所有来源中搜索推荐意见
  - 检索前 10 个相关片段
  - 综合："论文 A 推荐 X [引用]。
            论文 B 推荐 Y [引用]。
            二者在 Z 上存在差异。"

你：[得到一份综合答案]
    [若需追问，请改用 Chat]
```

**最适合：**
- 综合性、一次性问题
- 同时比较多份来源
- 希望由系统判断何为相关
- 需要多角度搜索的复杂问题
- 不需要来回对话

---

### 3. TRANSFORMATIONS（转换）- 基于模板的处理

**是什么：** 将可复用模板应用于某一来源，得到结构化输出。

**流程：**
```
1. 你定义转换（或选择预设）
   "提取：主要论点、方法论、局限性"

2. 你一次应用于一个来源
   （可对其他来源重复）

3. 对该来源：
   - 来源内容 + 转换提示词 → AI
   - 结果存为新洞察/笔记

4. 你得到
   - 结构化输出（主要论点、方法论、局限性）
   - 保存为笔记本中的笔记
```

**上下文管理：** 一次处理一个来源。

**可复用：** 同一模板可应用于不同来源（逐个进行）。

**注意**：当前一次处理一个来源。批量处理（同时处理多个来源）计划在后续版本提供。

**示例：**
```
你：定义转换
     "对每篇学术论文提取：
      - 主要研究问题
      - 所用方法论
      - 主要发现
      - 局限与空白
      - 建议的后续研究"

你：应用于论文 1

系统：
  - 对论文 1 运行转换
  - 结果存为新笔记

你：对论文 2、3 等应用同一转换

处理 10 篇论文后：
  - 你有 10 条格式一致的笔记
  - 适合撰写文献综述或对比
```

**最适合：**
- 从每个来源提取相同信息（重复运行）
- 创建格式一致的结构化摘要
- 构建分类洞察知识库
- 需要可复用、能逐来源应用的模板

## 决策树：用哪种工具？

```
你想做什么？

│
├─→ "我想就这一主题进行对话"
│   └─→ 对话是探索性还是固定？
│       ├─→ 探索性（会有后续追问）
│       │   └─→ 使用：CHAT
│       │
│       └─→ 固定（一问即结束）
│           └─→ 见下一问题
│
├─→ "我需要比较这些来源或得到综合答案"
│   └─→ 使用：ASK
│
├─→ "我想从每个来源提取相同信息（逐个）"
│   └─→ 使用：TRANSFORMATIONS（逐来源应用）
│
└─→ "我只想阅读与搜索"
    └─→ 使用：Search（文本或向量）
        或阅读你的笔记
```

## 并排对比

| 方面 | CHAT | ASK | TRANSFORMATIONS |
|--------|------|-----|-----------------|
| **用途** | 对话式探索 | 综合性问答 | 基于模板的提取 |
| **问题数量** | 多个（对话） | 一个 | 每来源一个模板 |
| **上下文控制** | 手动（你选择） | 自动（系统搜索） | 一次一个来源 |
| **是否对话？** | 是（支持追问） | 否（仅一问） | 否（单次操作） |
| **输出** | 自然对话 | 自然答案 | 结构化笔记 |
| **耗时** | 较快（来回） | 较长（综合） | 按来源计 |
| **最佳场景** | 探索且不确定 | 需要全貌 | 需要一致格式 |
| **模型速度** | 任意 | 建议较快 | 任意 |

## 工作流示例

### 示例 1：学术研究

```
目标：从 15 篇论文撰写文献综述

步骤 1：TRANSFORMATIONS
  - 定义："提取摘要、方法论、发现、相关性"
  - 应用于论文 1 → 得到结构化笔记
  - 应用于论文 2 → 得到结构化笔记
  - ……对全部 15 篇重复
  - 结果：15 条格式一致的笔记

步骤 2：阅读笔记
  - 现有一致性摘要

步骤 3：CHAT 或 ASK
  - Chat："帮我按主题整理这些"
  - Ask："这些论文有哪些共同方法论？"

步骤 4：撰写综述
  - 以转换为基础
  - 用 chat/ask 的洞察组织结构
```

### 示例 2：产品研究

```
目标：理解客户访谈反馈

步骤 1：添加来源（访谈记录）

步骤 2：ASK
  - "提到的前 10 大痛点是什么？"
  - 得到带引用的综合答案

步骤 3：CHAT
  - "能按严重程度帮我分组吗？"
  - 继续对话以确定优先级

步骤 4：TRANSFORMATIONS（可选）
  - 定义："提取：痛点、频次、提及者"
  - 逐份访谈应用
  - 得到用于分析的结构化数据
```

### 示例 3：政策分析

```
目标：比较政策文件

步骤 1：将所有政策文件添加为来源

步骤 2：ASK
  - "这些政策在气候措施上有何不同？"
  - 系统搜索全部文档，给出综合对比

步骤 3：CHAT（如需要）
  - "哪项政策最符合 X 目标？"
  - 讨论权衡取舍

步骤 4：导出笔记
  - 将 AI 回复保存为笔记用于报告
```

## 上下文管理：控制面板

三种模式均可控制 AI 所见内容。

### 在 CHAT 与 TRANSFORMATIONS 中
```
你选择：
  - 包含哪些来源
  - 每个来源的上下文级别：
    ✓ Full Content（发送完整文本）
    ✓ Summary Only（发送 AI 摘要，非全文）
    ✓ Not in Context（完全排除）

示例：
  论文 A：Full Content（精读分析）
  论文 B：Summary Only（背景）
  论文 C：Not in Context（机密）
```

### 在 ASK 中
```
上下文自动：
  - 系统搜索你的全部来源
  - 检索最相关片段
  - 发送给 AI

但你可：
  - 在特定笔记本内搜索
  - 按来源类型筛选
  - 用结果决定后续 Chat 的上下文
```

## 模型选择

各模式适用于不同模型：

### CHAT
- **任意模型**均可
- 快速模型（GPT-4o mini、Claude Haiku）：响应快，适合对话
- 强大模型（GPT-4o、Claude Sonnet）：推理更好，适合复杂主题

### ASK
- **建议较快模型**（因需多次搜索）
- 若需深度综合可使用强大模型
- 示例：GPT-4 用于策略规划，GPT-4o-mini 用于快速事实

### TRANSFORMATIONS
- **任意模型**均可
- 快速模型（批量处理更省成本）
- 强大模型（提取质量更好）

## 进阶：串联多种模式

可组合使用：

```
TRANSFORMATIONS → CHAT
  1. 用转换提取结构化数据
  2. 用 chat 讨论结果

ASK → TRANSFORMATIONS
  1. 用 Ask 理解重点
  2. 用 Transformations 从其余来源提取

CHAT → 存为笔记 → TRANSFORMATIONS
  1. 对话（Chat）
  2. 将优质回复存为笔记
  3. 以这些笔记为转换的上下文
```

## 摘要：何时使用哪一种

| 场景 | 使用 | 原因 |
|-----------|-----|-----|
| "我想用追问探索某一主题" | **CHAT** | 对话式，你控制上下文 |
| "我需要对一个复杂问题得到综合答案" | **ASK** | 自动搜索，综合回答 |
| "我想从每个来源得到一致摘要" | **TRANSFORMATIONS** | 模板复用，逐来源应用 |
| "我在比较两个特定来源" | **CHAT** | 仅选这 2 个，展开讨论 |
| "我需要按 X 标准对每个来源分类" | **TRANSFORMATIONS** | 从每个来源提取类别 |
| "我想了解全部来源的大局" | **ASK** | 自动综合搜索 |
| "我想构建知识库" | **TRANSFORMATIONS** | 从每个来源创建结构化笔记 |
| "我想迭代深化理解" | **CHAT** | 多轮提问， refine 思路 |

核心洞察：**不同问题需要不同工具。** Open Notebook 提供全部三种，因为研究很少只适用一种模式。
