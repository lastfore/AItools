# 添加来源 - 将内容导入笔记本

来源是研究的原始材料。本指南介绍如何添加不同类型的内容。

---

## 快速入门：添加第一个来源

### 方式 1：上传文件（PDF、Word 等）

```
1. 在笔记本中，点击「添加来源」
2. 选择「上传文件」
3. 从计算机中选择文件
4. 点击「上传」
5. 等待 30–60 秒完成处理
6. 完成！来源将出现在笔记本中
```

### 方式 2：添加网页链接

```
1. 点击「添加来源」
2. 选择「网页链接」
3. 粘贴 URL：https://example.com/article
4. 点击「添加」
5. 等待处理（通常比文件更快）
6. 完成！
```

### 方式 3：粘贴文本

```
1. 点击「添加来源」
2. 选择「文本」
3. 粘贴或输入内容
4. 点击「保存」
5. 完成！立即可用
```

### 方式 4：关键词搜索网页（SearXNG）

当管理员已部署并启用 **SearXNG**（自托管元搜索引擎）时，您可以在公网上用关键词发现页面，并批量添加为「网页链接」类来源。

```
1. 在笔记本中打开「添加来源」→「关键词搜索网页」
2. 输入关键词（可选语言、时间范围等）
3. 点击「搜索」
4. 勾选要添加的 URL（每批最多 50 个）
5. 选择笔记本与处理选项（嵌入、转换等）
6. 点击「添加所选」——每个 URL 会按普通网页链接来源抓取
```

这搜索的是**互联网**，不是您已导入的笔记本资料库。若要检索已收录内容，请使用笔记本内的 **搜索** 或「搜索」页面（知识库搜索）。

**前提：** SearXNG 服务已运行，且 API 环境变量中 `SEARXNG_ENABLED=true`。Windows 开发环境下，`.env` 启用后 `.\start-dev.ps1` 会自动启动 SearXNG。详见 [Docker Compose 安装](../../docs/1-INSTALLATION/docker-compose.md#adding-searxng-internet-keyword-search) 与 [环境变量参考](../../docs/5-CONFIGURATION/environment-reference.md#searxng-internet-keyword-search)。

## 支持的文件类型

### 文档
- **PDF** (.pdf) — 支持最佳，包括带 OCR 的扫描 PDF
- **Word** (.docx, .doc) — 完整支持
- **PowerPoint** (.pptx) — 幻灯片转换为文本
- **Excel** (.xlsx, .xls) — 电子表格数据
- **EPUB** (.epub) — 电子书文件
- **Markdown** (.md, .txt) — 纯文本格式
- **HTML** (.html, .htm) — 网页文件

**文件大小限制：** 最大约 100MB（因系统而异）

**处理时间：** 10 秒至 2 分钟（取决于长度与文件类型）

### 音频与视频
- **音频**：MP3、WAV、M4A、OGG、FLAC（约每小时 30 秒至 3 分钟）
- **视频**：MP4、AVI、MOV、MKV、WebM（约每小时 3–10 分钟）
- **YouTube**：支持直接 URL
- **播客**：RSS 订阅 URL

**自动转录**：音频/视频会自动转录为文本。需在设置中启用语音转文字（speech-to-text）。

### 网页内容
- **文章**：博客文章、新闻、Medium
- **YouTube**：完整视频或播放列表
- **在线 PDF**：直接 PDF 链接
- **新闻**：新闻网站文章

在「网页链接」部分**粘贴 URL** 即可。

### 不支持的内容
- 付费墙内容（WSJ、FT 等）— 无法提取
- 受密码保护的 PDF — 无法打开
- 纯图片文件 (.jpg, .png) — 扫描 PDF 除外（支持 OCR）
- 超大文件（>100MB）— 会超时

## 添加来源时系统会做什么

系统会自动执行四项操作：

```
1. 提取文本
   文件/URL → 可读文本
   （扫描 PDF 会进行 OCR）
   （若已启用，视频会进行转录）

2. 分块
   长文本 → 约 500 词一段
   （便于搜索定位具体部分，而非整篇文档）

3. 创建嵌入
   每块 → 向量表示
   （支持语义/概念搜索）

4. 索引与存储
   全部内容 → 数据库
   （可供搜索与检索）
```

**可用时间：** 进度条完成后，来源立即可用。嵌入在后台创建。

## 各类型的分步说明

### PDF

**最佳实践：**
```
清晰 PDF：
  1. 上传 → 完成
  2. 处理时间：约 30–60 秒

扫描/图片 PDF：
  1. 同样方式上传
  2. 系统自动检测并使用 OCR
  3. 处理时间：约 2–3 分钟
  4. （因 OCR 开销更高）

大型 PDF（50+ 页）：
  1. 考虑拆分为较小文件
  2. 或直接上传（系统可处理）
  3. 处理时间随大小增加
```

**常见问题：**
- 「无法提取文本」→ PDF 损坏或有复制保护
- 解决方法：尝试用 Adobe 打开。若无法打开，PDF 可能受保护。

### 网页链接 / 文章

**最佳实践：**
```
1. 从浏览器复制完整 URL：https://example.com/article-title
2. 粘贴到「网页链接」
3. 点击添加
4. 等待提取

处理时间：通常 5–15 秒
```

**适用情况：**
- 标准网页文章
- 博客文章
- 新闻文章
- 维基百科页面
- Medium 文章
- Substack 文章

**不适用情况：**
- Twitter 串（不稳定）
- 付费墙文章（无法访问）
- 重度 JavaScript 网站（内容无法提取）

**专业提示：** 若无法提取，可复制文章文本并以「文本」方式粘贴。

### 音频文件

**最佳实践：**
```
1. 确保在设置中已启用语音转文字
2. 上传 MP3、WAV 或 M4A 文件
3. 系统自动转录为文本
4. 处理时间：约每 5 分钟音频需 1 分钟

示例：
  - 1 小时播客 → 约 12 分钟处理
  - 10 分钟录音 → 约 2 分钟处理
```

**音质很重要：**
- 清晰音频：转录快
- 模糊/嘈杂音频：转录慢、准确度低
- 背景噪声：上传前尽量降低

**提示：** 若音质差，AI 可能误解内容。必要时可手动修正转录。

### YouTube 视频

**最佳实践：**
```
两种添加方式：

方式 1：直接 URL
  1. 复制 YouTube URL：https://www.youtube.com/watch?v=...
  2. 粘贴到「网页链接」
  3. 点击添加
  4. 系统提取字幕（如有）及转录

方式 2：播放列表
  1. 粘贴播放列表 URL
  2. 系统将所有视频添加为独立来源
  3. 每个视频单独处理
  4. 耗时更长（多个视频）
```

**提取内容：**
- 字幕/说明（如有）
- 转录（若无字幕）
- 基本元数据（标题、频道、时长）

**处理时间：**
- 10 分钟视频：约 2–3 分钟
- 1 小时视频：约 10–15 分钟

### 文本 / 粘贴内容

**最佳实践：**
```
1. 添加来源时选择「文本」
2. 粘贴或输入内容
3. 系统立即处理
4. 无需等待

适用于：
  - 需要引用的笔记
  - 书籍摘录
  - 手头已有的转录
  - 快速研究片段
```

## 上下文管理：来源如何被使用

您可控制 AI 如何访问来源：

### 三个级别（用于对话）

**完整内容：**
```
AI 可见：完整来源文本
成本：100% token
适用：详细分析、需要精确引用
示例：「仔细分析这篇方法论论文」
```

**仅摘要：**
```
AI 可见：AI 生成的摘要（非全文）
成本：约全文 token 的 10–20%
适用：背景材料、参考上下文
示例：「作为背景使用，但聚焦主要来源」
```

**不在上下文中：**
```
AI 可见：无（已排除）
成本：0 token
适用：机密、不相关或已归档内容
示例：「保留在笔记本中，但不在本次对话中使用」
```

### 如何设置上下文（在对话中）

```
1. 进入对话
2. 点击「选择上下文来源」
3. 对每个来源：
   - 开关：包含/排除
   - 选择级别（完整/摘要/排除）
4. 点击「保存」
5. 对话将使用这些设置
```

## 常见错误

| 错误 | 后果 | 修复方法 |
|---------|--------------|-----------|
| 一次上传 200 个来源 | 系统变慢、处理停滞 | 每次添加 10–20 个，等待处理完成 |
| 所有来源使用完整内容 | token 用量激增、成本高 | 背景材料使用「摘要」或「排除」 |
| 添加超大 PDF 未拆分 | 处理慢、搜索结果不精确 | 考虑将大型 PDF 按章节拆分 |
| 忘记设置来源标题 | 难以区分相似来源 | 上传后立即用描述性标题重命名 |
| 不添加标签 | 日后难以查找与整理 | 立即添加标签：「主要」「背景」等 |
| 单一来源混用多种语言 | 转录/嵌入质量下降 | 每种语言使用独立来源 |
| 重复添加同一来源 | 占用空间、造成混乱 | 只添加一次；在多个对话/笔记本中复用 |

## 处理状态与故障排除

### 状态指示含义

```
🟡 处理中
  → 正在提取并生成嵌入
  → 根据大小等待 30 秒至 3 分钟
  → 暂勿在对话中使用

🟢 就绪
  → 已处理且可搜索
  → 可立即在对话中使用
  → 可应用转换

🔴 错误
  → 出现问题
  → 常见原因：
    - 不支持的文件格式
    - 文件过大或损坏
    - 网络超时

⚪ 不在上下文中
  → 已添加但在对话中排除
  → 仍可搜索，不会发送给 AI
```

### 常见错误与解决方案

**「不支持的文件类型」**
- 尝试上传了列表外的格式（如 `.webp` 图片）
- 解决方法：转换为支持格式（文档用 PDF，音频用 MP3）

**「处理超时」**
- 文件非常大（>100MB）或音频很长
- 解决方法：拆分为较小部分或重新上传

**「转录失败」**
- 音频质量过差或未检测到语言
- 解决方法：以更好质量重新录制，或手动粘贴文本转录

**「网页链接无法提取」**
- 网站阻止自动访问或使用 JavaScript 渲染内容
- 解决方法：复制文章文本并以「文本」方式粘贴

## 获得最佳结果的提示

### PDF
- 清晰、数字版 PDF 效果最佳
- 如有复制保护，在合法前提下移除
- 扫描 PDF 可用但耗时更长

### 网页文章
- 使用包含域名的完整 URL
- 避免大量 Cookie/弹窗的网站
- 若提取失败，改为复制粘贴文本

### 音频
- 清晰、录制良好的音频转录更佳
- 尽量去除背景噪声
- YouTube 视频通常自带良好转录

### 大型文档
- 考虑拆分为较小来源
- 搜索结果更精确
- 较小片段处理更快

### 整理
- 为来源使用清晰名称（勿用「document_2.pdf」）
- 上传后立即添加标签
- 复杂文档添加描述
