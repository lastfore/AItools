# 引用 - 核实并信任 AI 回复

引用将 AI 回复与来源材料关联。本指南介绍如何使用与核实引用。

---

## 引用的重要性

Open Notebook 中每条 AI 生成回复均包含对您来源的引用。这使您可以：

- **核实主张** - 确认 AI 确实阅读了其所声称的内容
- **查找原文语境** - 查看引文周围的完整段落
- **发现幻觉** - 识别 AI 编造的内容
- **建立可信度** - 笔记具有可追溯来源

---

## 快速入门：使用引用

### 阅读引用

```
AI 回复：
「研究发现准确率为 95% [1]，采用所提方法。」

[1] = 点击查看来源

点击后：
→ 打开来源文档
→ 高亮相关章节
→ 可核实主张
```

### 请求更准确的引用

若回复缺少引用，可询问：

```
「请为该主张引用具体页码或章节。」
「文档中何处提到这一点？」
「能否引用原文？」
```

## 引用如何工作

### 自动生成

当 AI 引用您的来源时，引用会自动生成：

```
1. AI 分析您的问题
2. 检索相关来源片段
3. 生成带行内引用的回复
4. 将引用链接到原始来源位置
```

### 引用格式

```
行内格式：
「研究者得出结论 X [1] 与 Y [2]。」

参考文献列表：
[1] 论文标题 - 第 3.2 节
[2] 报告名称 - 第 15 页

可点击：每个 [数字] 链接到来源
```

## 核实引用

### 核实工作流

```
步骤 1：阅读 AI 回复
        「模型达到 95% 准确率 [1]」

步骤 2：点击引用 [1]
        → 打开来源文档
        → 显示相关段落

步骤 3：核实主张
        来源是否确实写 95%？
        语境是否正确？
        是否遗漏细微差别？

步骤 4：信任或更正
        ✓ 准确 → 使用该洞察
        ✗ 错误 → 要求 AI 更正
```

### 检查项

| Check | Why |
|-------|-----|
| **精确数字** | AI 有时会四舍五入或记错 |
| **语境** | 引文脱离语境可能含义不同 |
| **归属** | 是来源的主张还是其引用的他人？ |
| **完整性** | AI 是否遗漏重要限定？ |

## 各功能中的引用

### 对话引用

```
上下文：您选择的来源
引用：回复中使用的片段
核实：点击查看原文
保存：保存为笔记时保留引用
```

### Ask 功能引用

```
上下文：自动搜索所有来源
引用：综合多个来源
核实：各来源分别链接
质量：通常比对话更全面
```

### 转换引用

```
上下文：正在转换的单一来源
引用：指向原始文档
核实：将输出与来源对照
用途：需要结构化提取时
```

## 保存引用

### 在笔记中

将 AI 回复保存为笔记时，引用会保留：

```
原始回复：
「根据论文 [1]，该方法通过……工作」

保存的笔记包含：
- 文本
- 引用链接
- 来源文档引用
```

### 导出

导出时引用仍有效：

| Format | Citation Behavior |
|--------|-------------------|
| **Markdown** | 链接保留为 `[text](link)` |
| **Copy/Paste** | 纯文本带参考编号 |
| **PDF** | 可点击引用（若支持） |

## 引用质量提示

### 获得更好引用

**问题要具体：**
```
差：「关于 X 说了什么？」
好：「第 15 页关于 X 说了什么？请直接引用。」
```

**要求引用格式：**
```
「每个主张附页码。」
「引用具体章节，而非仅文档名。」
```

**使用完整内容上下文：**
```
仅摘要 → 引用不够精确
完整内容 → 可精确引用
```

### 缺少引用时

| Situation | Cause | Solution |
|-----------|-------|----------|
| 无引用 | AI 使用一般知识 | 问：「仅基于我的来源回答」 |
| 引用模糊 | 来源非完整内容 | 更改上下文级别 |
| 引用错误 | AI 混淆来源 | 要求用引文核实 |

## 常见问题

### 「引用与主张不符」

```
问题：AI 说 X，来源说 Y

原因：
- AI 转述错误
- AI 混淆多个来源
- 来源被断章取义

解决：
1. 点击引用查看原文
2. 记录差异
3. 问 AI：「来源写的是 Y 而非 X，请更正。」
```

### 「找不到引用章节」

```
问题：引用链接未显示相关文本

原因：
- 来源分块与预期不同
- 信息分散在多个章节
- 处理遗漏部分内容

解决：
1. 在来源内搜索关键词
2. 请 AI 提供更具体位置
3. 必要时重新处理来源
```

### 「完全没有引用」

```
问题：AI 回复无来源参考

原因：
- 来源不在上下文中
- 问题要求意见/一般知识
- 模型未找到相关内容

解决：
1. 检查上下文设置
2. 改述：「基于我的来源，……」
3. 添加更多相关来源
```

## 最佳实践

### 研究诚信

1. **始终核实重要主张** - 不要盲目信任 AI
2. **检查语境** - 断章取义可能误导
3. **注意局限** - AI 可能遗漏细微差别
4. **保留来源访问** - 勿删除已引用的来源

### 学术工作

1. **对将引用的文档使用完整内容**
2. **要求具体页码**
3. **与原始来源交叉核对**
4. **记录核实过程**

### 专业使用

1. **分享前核实** - 检查客户将看到的主张
2. **保留引用链** - 保存带来源链接的笔记
3. **保持透明** - 注明洞察由 AI 辅助生成

```
引用 = 您的核实系统

使用方法：
1. 阅读 AI 回复

引用失效时：

```
