## 各任务对应的功能

```
任务："我想通过追问探索某个主题"
→ 使用：对话（添加来源、选择上下文、进行对话）

任务："我想要一个全面的答案"
→ 使用：搜索 / Ask（系统自动查找相关内容）

任务："我想从多个来源提取相同信息"
→ 使用：转换（定义模板、应用于全部来源）

任务："我想为所有来源生成摘要"
→ 使用：转换（使用内置摘要模板）

任务："我想以音频形式分享研究成果"
→ 使用：播客（创建说话人、生成节目）

任务："我想找到记得的那句引文"
→ 使用：搜索 / 文本搜索（关键词匹配）

任务："我在探索某个概念，但不知道确切用词"
→ 使用：搜索 / 向量搜索（语义相似度）

任务："我需要添加或更改 AI 提供商 API 密钥"
→ 使用：设置 / API 密钥（无需编辑文件即可配置提供商）
```

## 快速入门清单：前 15 分钟

**步骤 1：创建笔记本（1 分钟）**
- 名称：具有描述性的名称（如「Q1 市场研究」「AI 安全论文」等）
- 描述：用 1–2 句话说明研究主题
- 这是您的研究容器

**步骤 2：添加第一个来源（3 分钟）**
- 任选其一：PDF、网页链接或文本
- 按照 [添加来源](adding-sources.zh.md) 操作
- 等待处理完成（通常 30–60 秒）

**步骤 3：就此进行对话（3 分钟）**
- 进入对话
- 选择您的来源（将上下文设为「完整内容」）
- 提出一个简单问题：「主要观点是什么？」
- 查看带引用的 AI 回复

**步骤 4：将洞察保存为笔记（2 分钟）**
- 回复满意？点击「保存为笔记」
- 使用有意义的名称（如「来源 X 的主要观点」）
- 您已捕获一条洞察

**步骤 5：进一步探索（6 分钟）**
- 再添加一个来源
- 就两者一起进行对话
- 提出一个比较类问题
- 通过追问澄清细节

**完成！** 您已使用核心工作流：笔记本 → 来源 → 对话 → 笔记

## 应避免的常见错误

| 错误 | 问题 | 修复方法 |
|---------|---------|-----|
| 将所有内容放入一个笔记本 | 项目之间无法隔离 | 为不同主题创建独立笔记本 |
| 期望 AI 了解您的背景 | 问题得到泛泛回答 | 在对话上下文中说明研究重点 |
| 忘记引用来源 | 无法核实主张 | 点击引用查看来源片段 |
| 用对话处理一次性问题 | 比 Ask 更慢 | 全面问答用 Ask，探索用对话 |
| 添加超大 PDF 且未分块 | 处理慢、搜索效果差 | 如可能，拆分为多个较小来源 |
| 所有对话使用相同上下文 | 成本高、焦点分散 | 为每次对话调整上下文级别 |
| 忽略向量搜索 | 只能找到精确关键词 | 使用向量搜索进行概念探索 |
