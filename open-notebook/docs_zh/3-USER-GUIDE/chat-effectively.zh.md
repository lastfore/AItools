# 高效对话 - 与研究材料进行对话

对话是您进行探索性提问与来回交流的主要工具。本指南介绍如何有效使用。

---

## 快速入门：第一次对话

```
1. 进入笔记本
2. 点击「Chat」
3. 选择要包含的来源（上下文）
4. 输入问题
5. 点击「发送」
6. 阅读回复
7. 提出追问（上下文保持不变）
8. 重复直至满意
```

仅此而已！但要**用好**，需要理解上下文如何工作。

---

## 上下文管理：优质对话的关键

上下文控制**AI 被允许看到的内容**。这是最重要的控制项。

### 三个级别说明

**完整内容**
- AI 可见：完整来源文本
- 成本：来源每 1K token 消耗 100 token
- 最适合：详细分析、精确引用
- 示例：「仔细分析这篇研究论文」

```
您设置：论文 A → 完整内容
AI 可见：论文 A 的每个字
AI 可以：引用具体句子、注意细微差别
结果：精确、详细的回答（成本更高）
```

**仅摘要**
- AI 可见：AI 生成的约 200 词摘要（非全文）
- 成本：约为完整内容的 10–20%
- 最适合：背景材料、参考上下文
- 示例：「作为背景使用，聚焦主要论文」

```
您设置：论文 B → 仅摘要
AI 可见：浓缩摘要、要点
AI 可以：引用主要观点但非细节
结果：更快、更便宜的回答（精度降低）
```

**不在上下文中**
- AI 可见：无
- 成本：0 token
- 最适合：机密、不相关或已归档内容
- 示例：「保留在笔记本中，但本次不使用」

```
您设置：论文 C → 不在上下文中
AI 可见：无（完全排除）
AI 可以：永不引用
结果：无成本、该来源无隐私风险
```

### 设置上下文（分步）

```
1. 点击「Select Sources」
   （显示笔记本中所有来源列表）

2. 对每个来源：
   □ 复选框：包含或排除

   级别下拉：
   ├─ Full Content
   ├─ Summary Only
   └─ Excluded

3. 检查选择
   示例：
   ✓ 论文 A（完整内容）- 「主要焦点」
   ✓ 论文 B（仅摘要）- 「背景」
   ✓ 论文 C（排除）- 「保持私密」
   □ 论文 D（未包含）- 「不相关」

4. 点击「Save Context」

5. 对话将使用这些设置
```

### 上下文策略

**策略 1：极简**
- 主要来源：完整内容
- 其余：排除
- 结果：聚焦、低成本、精确

```
适用于：
  - 深入分析单一来源
  - 预算有限
  - 需要聚焦的回答
```

**策略 2：全面**
- 所有来源：完整内容
- 结果：考虑全部上下文，成本高

```
适用于：
  - 全面分析
  - 预算充足
  - 希望 AI 看到一切
```

**策略 3：分层**
- 主要来源：完整内容
- 次要来源：仅摘要
- 背景/参考：排除
- 结果：成本与质量平衡

```
适用于：
  - 重要与参考材料混合
  - 希望全面但不过贵
  - 最常见策略
```

**策略 4：隐私优先**
- 敏感文档：排除
- 公开研究：完整内容
- 结果：永不发送机密数据

```
适用于：
  - 公司机密材料
  - 个人敏感数据
  - 符合数据保护要求
```

## 提出有效问题

### 差问题与好问题

**差问题**
```
「你怎么看？」

问题：
- 过于模糊（关于什么？）
- 无上下文（在分析什么？）
- 无法核实回答（引用什么？）

结果：泛泛、肤浅的回答
```

**好问题**
```
「根据论文的方法论部分，
作者承认的三个主要局限是什么？
请引用各局限所在的页码。」

优点：
- 明确所需内容
- 范围清晰（方法论部分）
- 要求引用
- 需要深入阅读

结果：精确、可核实、有用的回答
```

### 有效的问题模式

**事实性问题**
```
「论文关于 X 说了什么？」
「作者是谁？」
「哪年发表？」

结果：简单、带引用的事实性回答
```

**分析性问题**
```
「这种方法与传统方法有何不同？」
「该论证的主要假设是什么？」
「作者为何选择这种方法论？」

结果：更深入思考、比较、评析
```

**综合性问题**
```
「这两份来源处理问题的方式有何不同？」
「三篇论文的共同主题是什么？」
「若结合这些方法，会得到什么？」

结果：跨来源洞察、关联
```

**可操作性问题**
```
「这项研究的实际意义是什么？」
「如何将这些发现应用于我们的情况？」
「下一个合理的研究方向是什么？」

结果：实用、前瞻的回答
```

### SPECIFIC 公式

好问题包含：

1. **SCOPE（范围）** - 分析什么？
   「在这篇研究论文中……」
   「看这三篇文章……」
   「根据您的经验……」

2. **SPECIFICITY（具体性）** - 确切要什么？
   「……方法论……」
   「……主要发现……」
   「……建议的后续步骤……」

3. **CONSTRAINT（约束）** - 有何限制？
   「……用 3 个要点……」
   「……引用页码……」
   「……比较这两种方法……」

4. **VERIFICATION（可验证性）** - 如何核对？
   「……附具体引文……」
   「……引用来源……」
   「……链接到相关章节……」

**示例：**
```
差：「transformer 怎么样？」
好：「在这篇机器学习研究论文中，
      用 2–3 句话解释 transformer 架构，
      然后引用描述注意力机制的页码。」
```

## 追问（对话的真正力量）

对话的优势在于对话。您提问、获得回答、继续提问。

### 在回复基础上延伸

```
第一个问题：
「主要发现是什么？」

AI：「研究表明 X [引用]」

追问：
「这与 Y 研究相比如何？」

AI：「主要差异是 Z [引用]」

下一个问题：
「您认为这一差异为何重要？」

AI：「因为它影响 A、B、C [说明]」
```

### 迭代加深理解

```
第 1 轮：概览
「这份来源讲什么？」

第 2 轮：细节
「最重要的部分是什么？」

第 3 轮：比较
「这与我对 X 的笔记有何关系？」

第 4 轮：应用
「我应如何处理这些信息？」
```

### 转换方向

```
上下文相同，但提出新问题：

问题 1：「方法论是什么？」
问题 2：「局限有哪些？」
问题 3：「伦理影响如何？」
问题 4：「还有谁做过类似工作？」

同一对话中复用上下文。
```

### 轮次间调整上下文

```
第 3 个问题后，您意识到：
「我需要另一份来源的更多上下文」

1. 点击「Adjust Context」
2. 添加新来源或更改上下文级别
3. 对话历史保留
4. 在新上下文中继续提问
```

## 引用与核实

引用用于核实 AI 回答是否准确。

### 理解引用

```
带引用的 AI 回复：
「论文报告准确率为 95% [见第 12 页]」

含义：
✓ 「95% 准确率」来自第 12 页
✓ 可阅读第 12 页核实
✓ 若第 12 页未如此表述，则 AI 产生了幻觉
```

### 请求更准确的引用

```
若回复无引用：

问：「请为该主张引用具体页码」
或：「请说明您在哪里找到该信息」

AI 将：
- 找到引用
- 提供页码
- 展示来源
```

### 核实工作流

```
1. 从对话获得回答
2. 检查引用（哪个来源？哪一页？）
3. 点击引用链接（如有）
4. 查看来源中的实际文本
5. 是否确实如 AI 所述？

若是：很好，可使用该回答
若否：AI 产生幻觉，要求更正
```

## 常见对话模式

### 模式 1：深入单一来源

```
1. 设置上下文：一个来源（完整内容）
2. 问题 1：概览
3. 问题 2：主要论点
4. 问题 3：论点证据
5. 问题 4：局限
6. 问题 5：后续步骤

结果：完整理解单一来源
```

### 模式 2：比较分析

```
1. 设置上下文：2–3 个来源（均为完整内容）
2. 问题 1：各来源对 X 怎么说？
3. 问题 2：一致之处？
4. 问题 3：分歧之处？
5. 问题 4：哪种方法更强？

结果：理解差异与权衡
```

### 模式 3：研究探索

```
1. 设置上下文：多个来源（完整/摘要混合）
2. 问题 1：主要观点有哪些？
3. 问题 2：这些观点缺什么？
4. 问题 3：引发哪些问题？
5. 问题 4：下一步应研究什么？

结果：理解领域全景与空白
```

### 模式 4：问题解决

```
1. 设置上下文：相关来源（完整内容）
2. 问题 1：问题是什么？
3. 问题 2：有哪些方法？
4. 问题 3：各方法利弊？
5. 问题 4：哪种最适合[我的情况]？

结果：基于研究的决策
```

## 成本优化

对话每次回复都消耗 token。高效使用方式：

### 减少 Token 用量

**最小化上下文**
```
选项 A：所有来源，完整内容
  每次回复成本：5,000 token

选项 B：仅相关来源，仅摘要
  每次回复成本：1,000 token

节省：便宜 80%，对话相同
```

**更短的问题**
```
冗长：「能否请分析这篇论文的方法论部分
         并详细说明作者做了什么？」

简洁：「用 2–3 点概括方法论。」

节省：每次回复约 20–30%
```

**使用更便宜的模型**
```
GPT-4o：每 100 万输入 token $0.15
GPT-4o-mini：每 100 万输入 token $0.03
Claude Sonnet：每 100 万输入 token $0.90

对话：Mini/Haiku 通常足够
深度分析：Sonnet/Opus 值得成本
```

### 预算策略

**探索预算**
- 使用便宜模型
- 广泛上下文（了解全景）
- 简短问题
- 结果：低成本、良好概览

**分析预算**
- 使用强大模型
- 聚焦上下文（仅主要来源）
- 详细问题
- 结果：成本较高、深度洞察

**综合预算**
- 最终综合使用强大模型
- 多个来源（完整内容）
- 复杂比较问题
- 结果：昂贵但有价值

## 对话问题故障排除

### 回复质量差

| Problem | Cause | Solution |
|---------|-------|----------|
| 泛泛回答 | 问题模糊 | 具体化（见问题模式） |
| 缺少上下文 | 上下文不足 | 添加来源或改为完整内容 |
| 信息错误 | 来源不在上下文中 | 添加相关来源 |
| 产生幻觉 | 模型混淆 | 要求引用、核实主张 |
| 分析肤浅 | 模型不当 | 切换到更强大的模型 |

### 成本高

| Problem | Cause | Solution |
|---------|-------|----------|
| 单次回复昂贵 | 上下文过多 | 使用仅摘要或排除来源 |
| 大量追问 | 探索性对话 | 一次性全面问答用 Ask |
| 长对话 | 保留历史 | 归档旧对话、重新开始 |
| 大型来源 | 全文在上下文中 | 大文档使用仅摘要 |

## 最佳实践

### 对话前

- [ ] 添加所需来源
- [ ] 决定上下文策略（分层通常最佳）
- [ ] 选择模型（探索用便宜、分析用强大）
- [ ] 想好要问的问题

### 对话中

- [ ] 提出具体问题（使用 SPECIFIC 公式）
- [ ] 对事实性主张检查引用
- [ ] 对不清楚处追问
- [ ] 需要不同来源时调整上下文

### 对话后

- [ ] 将优质回复保存为笔记
- [ ] 完成后归档对话
- [ ] 整理笔记供日后参考
- [ ] 在其他功能中使用洞察（Ask、转换、播客）

## 何时用对话 vs Ask

**使用对话当：**
- 需要来回交流
- 正在探索主题
- 将提出多个相关问题
- 希望在对话中调整上下文
- 尚不确定确切需求

**使用 Ask 当：**
- 只有一个具体问题
- 需要全面答案
- 希望系统自动搜索
- 只需一次回复，非对话
- 希望将 token 主要用于搜索

## 摘要：对话即交流

对话与直接问 ChatGPT 有本质区别：

| Aspect | ChatGPT | Open Notebook Chat |
|--------|---------|-------------------|
| **来源控制** | 无（使用训练数据） | 您控制可见来源 |
| **成本控制** | 按 token | 按 token，但上下文由您选择 |
| **迭代** | 可用 | 可用，来源可动态变化 |
| **引用** | 常编造 | 绑定您的来源（可核实） |
| **隐私** | 数据发给 OpenAI | 数据保留本地（除非您选择否则） |

关键洞察：**对话是检索增强生成（RAG）。** AI 仅看到您放入上下文的内容。您控制对话与信息流。

因此对话对研究很有价值。您不仅在和 AI 说话；您在与研究材料本身对话。
